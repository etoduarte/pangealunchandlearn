The Problem

Creative is king. But Meta hides the data.

Creative has become the single most important lever in Meta ad performance. The algorithm optimizes for engagement, and engagement lives in the creative. But here's the problem: Meta's reporting structure makes creative performance nearly invisible — because Meta reports on a media buying basis, not a creative basis.

This creates fragmentation in two directions:

Same file, different line items. Place the exact same JPG across a geo split test, for account creation vs. first transfers, or when relaunching a campaign — Meta creates a new line item for each. The same creative appears in 5+ rows with no connection between them.
Same concept, different files. The creative idea is identical but the files differ: a price comparison ad updated for compliance, the same ad in English vs. Spanish, the same concept localized for Mexico, Colombia, Guatemala with different currencies. Meta treats these as completely unrelated assets.
Mexico · vs Xoom · MXN
Colombia · vs Remitly · COP

Same concept. Different country, competitor, currency, slang. Meta sees two unrelated assets.

You need both views. Roll up to the concept level — "How is the Podium concept performing across the board?" — AND slice into the metadata — "Podium works in Mexico but not Colombia." Meta gives you the second view, badly fragmented. The first view doesn't exist.

The result: an incomplete picture that's painfully slow to assemble.

The Hack

Download everything. Pivot on ad name.

The workaround everyone uses: export the raw data from Ads Manager, bring it into Excel, and pivot on ad name. Five campaign rows collapse into one creative row. You finally see aggregated numbers — total spend, total impressions, true CTR.

It works. But it's hours of manual VLOOKUP and pivot table work. And the data is already stale by the time you finish. Do this for every creative, every week, and you've built a full-time job that produces yesterday's insights.

The Solution

The creative layer Meta is missing.

The Pangea Marketing Suite solves both problems:

1
Pulls data from Facebook automatically — no more manual CSV exports.
2
Parses metadata — country, language, concept — extracted and structured from the file name.
3
Creates first-class objects that don't exist in Meta's model. "Creative" = same file. "Concept" = same idea, different files. Both sit alongside Campaigns, Ad Sets, and Ads.
4
Aggregation engine — combine any mix of objects for display and benchmarking. One concept across all countries? One click.
5
Built-in reporting — visual, fast, no Excel required.

The key insight: the tool doesn't throw away Meta's reporting structure. It adds the creative layer on top that Meta is missing. You get both views — the media buying view and the creative view — in the same place.

Plugins

Agents need instructions — plugins are the playbook. Each plugin teaches Claude Code how to behave for a specific job. These are real, community-built plugins anyone can install. My pipeline uses custom ones (Cal, Lisa, Ralph Wiggum), but the ecosystem has hundreds.

The Process
1

BRD Generation

Tool: Cal

This is the brainstorm phase. I created a Claude plugin called Cal that has a number of skills — one of them holds a virtual "meeting." While in the meeting, Claude doesn't code. This lets me rant, zig, zag, and explore different areas safely. At the end, we use an agent to clean up the meeting minutes and produce a BRD that we take into the next step.

github.com/etoduarte/cal-plugin
2

User Stories

Tool: Lisa

Lisa is a third-party plugin that takes the BRD and breaks it into user stories — development-ready specifications written from the user's perspective. It also asks you about any gaps or non-obvious issues that might pop up during implementation.

github.com/blencorp/lisa
3

Adversarial Review

Tool: Bart

Pressure-test the requirements before any code gets written. A dedicated adversarial review agent whose entire job is to disagree — find gaps, challenge assumptions, surface edge cases. This is a fence: nothing moves to implementation until it passes. I mediate the disagreements, deciding what matters and what doesn't.

Tips: Ask the AI to "explain this to me as if I'm a marketer" (or whatever your role is). Ask it to explain the tradeoffs of each decision — not just what it recommends, but what you'd be giving up.
4

Implementation

Tool: Ralph Wiggum

The implementation phase — actually writing the code. Ralph Wiggum is a Claude Code plugin that went viral on X for its unconventional approach: it uses a continuous loop where the AI writes code, runs tests, and iterates — all while working from the Bart-approved user stories. It builds features one at a time, and because it has clear specs from the previous steps, it stays focused instead of guessing.

5

QA & Code Protection

Tool: Sacred Code Agent

Guard what's already working. Uses deliberately unusual shared language in code comments to mark protected areas. Standard comments like // DO NOT MODIFY get ignored by coding AIs — they've seen millions of them in training data. Sacred Code uses dramatic, non-standard language that creates friction. The protection works because it's weird.

Quick tips that work in any Claude plan

Show, don't describe. Paste 3-5 examples of what you want before asking Claude to create something new. More effective than any written instruction.
